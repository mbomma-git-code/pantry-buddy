# Write Documentation Skill 💎📝

# Description
Use when the user wants to document a
API, endpoint, or component. Creates clear,
concise docs with usage examples.

# Instructions
1. Read the source code to understand its
purpose.
2. Identify the public API (exports,
parameters, etc.)
3. Generate documentation including:
- Brief description (one line)
- Detailed explanation (one paragraph)
- Parameters table with types and
descriptions
- Any thrown errors or edge cases
4. Use 30Doc format for inline code
5. Use Markdown for standalone doc
files
6. Create a mermaid diagram of the flow
